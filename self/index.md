# GTD

## key ideas
* Do not keep everything in your brain, use external system. Less stuff in brain = less stress.
* Each task should be split in easy steps, "physical" actions.
* TODO lists should contain steps (aka actions), not abstract goals.
* `Projects` are goals, i.e. results that you'd like to get.

## method
1. Collect everything
    - every idea should be placed out of brain into external system as soon as it comes to brain
    - minimal number of "inbox" places
    - regular revision of inboxes
2. Handle
   ```
   Does it need any action? ----+ NO ----> Feature / Reference / Trash
    |
    +
   YES -----> Is it part of a Project? -----+ YES ----> create new / update existing
                          |
                          +
                         NO ----> Does it take > 2 min? -----+ NO -----> do it
                                               |
                                               +
   add to calendar (particular date) <-----   YES -----> delegate
                                               |
                                              \ /
                        add to NEXT list (do as soon as possible)
   ```
3. Review variations of results for `Projects`, pick one
4. Do it

## inboxes
* **Calendars** for everything that has time
* **Projects list** with steps and contexts (home/work/etc)
* **Next list**, tasks from here should be picked up as soon as previous is finished
* **Future list** for smth useful you want to do, but not now
* **References**, place for useful information
